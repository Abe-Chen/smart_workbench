# 第一章 产品顶层定义

## 1.1 产品边界

**一句话定义**

> 一台桌面平板，开机即用。用户对它说话，它把日程和备忘记下来、到点叫人；问它什么它就答。没有登录，没有账号，所有数据在本地。

**MVP 做什么**

| 模块 | 做的事 |
|---|---|
| 日程 | 创建 / 查询 / 修改 / 删除 / 完成 / 重复展开 / 11 档提醒触发 |
| 备忘 | 创建 / 查询 / 智能提醒 / 转日程 / 状态流转 |
| AI 对话 | 唤醒 → 听 → 答 → 写 → 提醒，全语音闭环，含信息卡查询（天气/汇率/世界时钟/POI/新闻） |

**MVP 不做（明确砍）**

邮件、会议纪要、微信消息、客户资料、企业系统对接、多端同步、多账号、家庭模式、闲聊陪伴、久坐提醒、每日简报。

> ⚠️ 「不做」不是「以后不做」，而是「现在评审验收只看 MVP 内的，外面的功能即使代码里有也不算 MVP 完成度」。

## 1.2 目标用户画像

**45-60 岁商务精英**

| 维度 | 特征 |
|---|---|
| 一天典型 | 8-12 个会议 / 客户拜访 / 电话，时间被切得很碎 |
| 痛点 | 怕漏事，但记本子来不及；用手机日历又嫌打字慢、字小 |
| 设备使用 | 习惯桌上摆一台平板，看到的优先级高于手机 |
| 心智模型 | 信「秘书」，不信「机器人」 |
| 对 AI 的预期 | 像个能听话的助理，不是聊天玩具 |

**不是这类用户**：技术爱好者、年轻人尝鲜、Z 世代社交。

## 1.3 12 个核心场景（场景 → 需求 → 功能）

按用户视角写完整故事，再推导功能。

### S1 — 临时定会议
- **场景**：王总刚打完电话，问「明天下午 3 点跟我开个会」。用户对桌面平板说：「明天下午 3 点跟王总开会，提前半小时提醒我。」
- **需求**：记一条日程，标题+日期+时间+提醒一次说清，不要让我重复
- **功能**：助理识别意图 + 抽 4 个槽（标题/日期/时间/提醒）→ 弹确认卡 → 用户说「确认」→ 写库 + 注册系统通知 → TTS 回复 + 大卡消散
- **预期结束态**：数据库多一条 task，到提醒时间点系统级闹钟弹卡

### S2 — 查看明天有什么
- **场景**：用户结束晚饭，习惯性问「明天有什么安排？」
- **需求**：知道明天的全部日程，按时间排序，看一眼
- **功能**：助理识别查询意图（query_tasks_tool）→ 弹大卡显示明天日程列表 → TTS 摘要播报「明天有 3 个安排：…」
- **预期结束态**：列表卡持续 5 秒后自动消散，用户可点卡延长查看

### S3 — 改时间
- **场景**：客户说会议挪到 4 点。用户：「把刚才那个会议改到 4 点。」
- **需求**：不需要重新讲一遍标题，直接基于上下文改
- **功能**：助理走 update_task → 槽位继承 + 上下文找最近的会议 → 弹「这样改吗」确认 → 用户确认 → 改库 + 重发通知
- **预期结束态**：原通知取消，新通知按新时间触发

### S4 — 删
- **场景**：客户取消会议。用户：「那个会议取消。」
- **需求**：找到那条删了，别让我点
- **功能**：助理走 delete_task → 确认 → 删库 + 取消通知 → 5 秒内可点撤销
- **预期结束态**：task.status=deleted；撤销窗口 5 秒

### S5 — 记一条
- **场景**：张总说「让财务核 Q2 款项」。用户：「记一下，张总让财务核 Q2 的款项。」
- **需求**：先存下来，不一定有具体时间
- **功能**：助理走 create_memo → 弹「这条要记下来吗」→ 用户确认 → 存库
- **预期结束态**：memo 表多一条 active 状态备忘

### S6 — 记一条有时间的
- **场景**：用户：「下周要给客户发提案了，提醒我。」
- **需求**：备忘里有时间信息，到点提醒我
- **功能**：助理识别时间表述（「下周」=日精度）→ create_memo + 设 smartRemindEnabled + nextPromptAt → 到点弹紧急大卡「您之前记的…，处理了吗？」
- **预期结束态**：memo + 待触发提醒

### S7 — 查天气
- **场景**：用户出门前问「今天天气怎么样？」
- **需求**：当前位置当日天气，含温度、风、提醒
- **功能**：助理走 realtime_info 意图 → 豆包联网 → 解析成 weather 信息卡
- **预期结束态**：weather 卡显示 5 秒后消散

### S8 — 查汇率
- **场景**：用户：「美元今天多少？」
- **需求**：1 USD 兑 RMB 当前汇率
- **功能**：助理走 realtime_info → 豆包联网 → exchange 信息卡
- **预期结束态**：exchange 卡显示

### S9 — 查附近吃饭
- **场景**：用户中午想吃湘菜：「附近哪儿有好的湘菜？」
- **需求**：附近 1km 内的湘菜馆，含评分
- **功能**：助理走 local_search → get_user_location + 豆包联网 → poi 信息卡
- **预期结束态**：poi 卡显示候选 3-5 家

### S10 — 查新闻
- **场景**：用户：「最近有什么科技新闻？」
- **需求**：最近的科技领域要闻
- **功能**：助理走 realtime_info → 豆包联网 → news 信息卡
- **预期结束态**：news 卡显示头条

### S11 — 到点提醒
- **场景**：明天下午 3 点开会，到 2:30（提前半小时）系统闹钟触发
- **需求**：平板大屏弹起来，能看到是什么会议，能延 / 能关
- **功能**：flutter_local_notifications 触发 → assistant_controller.enqueueReminder → 弹 reminder 大卡 + TTS 播报「下午 3 点，跟王总开会」→ 三个按钮：我知道了 / 再延 10 分钟 / 关闭
- **预期结束态**：用户操作前大卡不消散；操作后写库 + 闹钟取消/重排

### S12 — 备忘转日程
- **场景**：用户之前记了「下周给客户发提案」，到下周一时助理弹提醒「这件事处理了吗？」用户：「转日程，下周三上午。」
- **需求**：把备忘变日程，时间补全
- **功能**：助理走 convert_memo_to_schedule → 弹「帮您记录到日程里吗」→ 确认 → 写 task + memo 状态改 converted
- **预期结束态**：task 多一条，memo.status=converted，linkedScheduleId 指向新 task

## 1.4 日程模块完整业务流程

### 1.4.1 日程能记什么（数据模型，已实现）

| 字段 | 取值 | 说明 |
|---|---|---|
| 标题 | 文本 | 必填 |
| 日期 | YYYY-MM-DD | 必填 |
| 全天 | 是 / 否 | 全天时不填时间 |
| 起止时间 | HH:MM ~ HH:MM | 非全天时必填 |
| 提醒 | 11 档 | 无 / 当天 9 点 / 中午 / 18 点 / 前一天 9 点 / 准时 / 提前 5/10/30 分钟 / 提前 1 小时 / 自定义 |
| 重复 | 4 档 | 不重复 / 每天 / 每周 / 每月 |
| 状态 | pending / completed / deleted | 软删除 |

### 1.4.2 创建流程（含全量异常）

**正常流（S1）**

```
听到「明天下午 3 点跟王总开会，提前半小时提醒我」
  ↓ ASR 识别 + NLU 语气词剥离
助理路由 → schedule_write 意图
  ↓ 抽取 4 槽：标题=跟王总开会 / 日期=2026-05-18 / 起=15:00 / 提醒=before30m
弹确认卡（顶部「没问题我就这样记了哈」+ 4 行摘要）
  ↓ 用户说「确认」或点确认按钮
写入 sqflite + 注册系统通知
  ↓
助理 TTS「好的，已经放到日程里了。明天 15:00 跟王总开会。」
  ↓ 5 秒后
大卡自动消散
```

**异常清单**

| 异常分支 | 助理表现 | 文案 |
|---|---|---|
| 全部信息都没听清 | 弹澄清卡 | 「可以。这个日程是什么，安排在什么时候？」 |
| 只缺标题 | 弹澄清卡 | 「这条日程叫什么？」 |
| 只缺日期 | 弹澄清卡 | 「这个安排在什么时候？」 |
| 只缺时间 | 弹澄清卡 + 给出兜底 | 「[日期]几点开始？也可以说"全天"。」 |
| 用户说取消 | 关闭确认卡 | 「好，这次先不记。」 |
| 用户在确认卡里说改 | 重新进抽槽流程 | 复用上次已有的槽位 |
| 数据库写入失败 | 弹错误卡 | 「这次没记上：[原因]。要不再来一次？」 |
| ASR 没听清 | 静默重试一次，再不行兜底 | 「您再说一次吧，标题和时间放一起说就行。」 |
| 用户带「啊嗯哈」语气词 | NLU 自动剥离 | 不打扰 |
| 用户带「你帮我」礼貌前缀 | NLU 剥离 | 不打扰 |

### 1.4.3 查询流程

| 维度 | 用户表达样例 | 助理响应 |
|---|---|---|
| 当日全部 | 今天有什么 | 列出全部，分上/下午分组 |
| 跨天范围 | 这周有什么 | 按日分组，「今天 2 项，明天 1 项…」 |
| 时段筛选 | 今天下午有空吗 | 只列下午（12:00-18:00） |
| 状态筛选 | 已完成的 / 已删的 | 带状态筛选 |
| 正在进行 | 我现在该做什么 | 当前时间正在进行的项 |
| 下一项 | 下一项是什么 | 时间序最近一项 |
| 跨日上限 | 最近 7 天 | 全量统计 + 按日分组 + 截断显示 |
| 空查询 + 下一项兜底 | 今天下午没安排吧 | 「今天下午没有安排。下一项是…」 |

### 1.4.4 修改流程

可改字段：标题、日期、时间、提醒、重复。

```
用户：「把那个会议改到 4 点。」
  ↓ 助理上下文找最近的会议（task_id）
弹「这样改吗」确认卡（显示前后对比：15:00 → 16:00）
  ↓ 用户确认
改库 + 取消原通知 + 注册新通知
  ↓
助理 TTS「好的，改好了。」
```

**异常**

| 异常分支 | 文案 |
|---|---|
| 没说要改哪个 | 「您想改哪一条？」 |
| 找不到唯一目标（多条匹配） | 列候选「您是说哪个？」 |
| 改完字段都没变 | 「这条没有变化哦。」 |

### 1.4.5 删除流程（含撤销）

```
用户：「把那个会议删了。」
  ↓
弹「这条要删吗」确认卡
  ↓ 用户确认
软删除 + 取消通知 + 显示 5 秒撤销条
  ↓ 用户点撤销 / 不操作
回滚 deleted 状态 / 永久删
```

**异常**

| 异常分支 | 文案 |
|---|---|
| 没说删哪个 | 「想删哪一条？我先查一下。」 |
| 多条匹配 | 列候选「您是说哪个？」 |
| 撤销窗口内点撤销 | 「好的，撤销了。」 |

### 1.4.6 完成流程

```
用户：「写周报标记完成。」
  ↓ 找到对应 task
直接执行（不需要二次确认，因为完成 = 标记，不丢数据）
  ↓
助理 TTS「已把「写周报」标记完成。刚才这一步可以撤销。」
  ↓ 5 秒撤销窗口
```

### 1.4.7 提醒触发流程

**注册环节**：创建 / 修改 task 时，按 reminderKey 计算触发时间，调用 `flutter_local_notifications.zonedSchedule` 注册系统级精准闹钟（`exactAllowWhileIdle`）。

**触发环节**：

```
系统闹钟到时间
  ↓ 进程内 → assistant_controller.enqueueReminder
若当前 TTS 正在播 → 入队等播完
若当前空闲 → 直接弹 reminder 大卡
  ↓
设 surfaceState=fullscreenAnswer + kind=reminder
TTS 播「[时间]，[标题]」
显示三按钮：我知道了 / 再延 10 分钟 / 关闭
  ↓ 不自动消散，等用户操作
```

**用户操作分支**

| 按钮 | 行为 |
|---|---|
| 我知道了 | 关卡，task 不变（重复任务下次还会响） |
| 再延 10 分钟 | 关卡，注册一个 10 分钟后的延迟通知 |
| 关闭 | 关卡，task 不变 |

**重复任务**：重复任务（每天/每周/每月）的提醒注册时按窗口（如未来 30 天）批量预排，到期自动响。

## 1.5 备忘模块完整业务流程

### 1.5.1 备忘能记什么（数据模型）

| 字段 | 取值 | 说明 |
|---|---|---|
| 标题 | 文本 | 必填 |
| 内容 | 长文本 | 选填 |
| 时间文本 | 文本（"下周"/"5 月底"） | 选填 |
| 时间精度 | 无 / 日 / 周 / 月 / 范围 | 自动识别 |
| 范围起止 | 日期 | 时间精度=范围时 |
| 状态 | active / converted / done / canceled | |
| 智能提醒 | 开 / 关 | 默认开 |
| 下次提醒时间 | 时间戳 | 由 memo_schedule_planner 算 |
| 关联日程 ID | int | 转日程后填 |

### 1.5.2 创建流程

```
用户：「记一下，张总让我核 Q2 的款项。」
  ↓ 助理路由 → memo_write
抽取槽：标题/内容/时间文本
弹「这条要记下来吗」
  ↓ 用户确认
写库
  ↓
助理 TTS「好的，记下来了。」
```

**异常**

| 异常分支 | 文案 |
|---|---|
| 标题不清楚 | 「需要加个备注吗？」 |
| 用户取消 | 「好，这次先不记。」 |
| 内容里有明确时间（如「下周三」） | 自动设智能提醒，不打扰用户 |

### 1.5.3 智能提醒流程

```
memo_schedule_planner 算下次提醒时间（基于 timePrecision）
  ↓ 写 nextPromptAt
flutter_local_notifications 注册精准闹钟
  ↓ 到点触发
助理弹紧急大卡「您之前记的：[标题]。这件事处理了吗？」
  ↓ 用户选项
处理了 → 状态改 done / 下周再问我 → 重设 nextPromptAt / 转日程 → 走转日程流程 / 关闭 → 不变
```

### 1.5.4 转日程流程（S12）

```
用户：「转日程，下周三上午。」
  ↓ 助理走 convert_memo_to_schedule
抽取日期 + 时间
弹「帮您记录到日程里吗」
  ↓ 用户确认
创建 task（继承备忘标题）+ memo.status=converted + 设 linkedScheduleId
  ↓
助理 TTS「好的，转过去了。」
```

**异常**

| 异常分支 | 文案 |
|---|---|
| 日期或时间没说 | 「日期或时间需要补充一下。」 |
| 已经转过了 | 「这条之前已经转过日程了。」 |

### 1.5.5 状态流转

```
active ─(用户「处理了」)─→ done
   │
   ├─(转日程)──────────→ converted
   │
   └─(用户取消)────────→ canceled
```

## 1.6 AI 对话模块完整业务流程（基础设施）

> 此节描述的是日程、备忘、信息查询所共用的对话基础设施。不是单独的 MVP 功能，是上述功能的**承载层**。

### 1.6.1 4 个 surface

| Surface | 触发条件 | 用途 |
|---|---|---|
| 球（永远在屏） | 任何时候 | 状态指示器（待命 / 听 / 思 / 答） |
| 顶部小浮窗 | ASR partial / 推送 | 实时显示用户在说的话 / 系统通知 |
| 全屏大卡 | 助理主动答复、确认、错误、提醒 | 唯一的"助理说话"载体 |
| 抽屉（上滑唤起） | 用户主动 | 看历史对话、看用户当前消息流 |

### 1.6.2 7 种大卡类型

| 类型 | 出现时机 | 是否自动消散 |
|---|---|---|
| plainText | 短答复 / 流式占位 | 5 秒 |
| infoCard | 天气/汇率/世界时钟/POI/新闻/日程查询 | 5 秒 |
| toolFeedback | 写库成功 / 删除成功 | 5 秒 |
| clarification | 缺字段追问 | 不消散，等用户回答 |
| confirm | 写入确认 | 不消散，等用户确认/取消 |
| reminder | 到点提醒 | 不消散，等用户操作 |
| error | 出错兜底 | 5 秒 |

### 1.6.3 流式呈现

豆包返回首字 → 立即设 surface=fullscreenAnswer + kind=plainText → 后续 delta 追加 → finish 切到最终 kind（infoCard / toolFeedback…）→ AnimatedSwitcher 平滑过渡。

### 1.6.4 等待期可控（MVP 重点补强）

10 秒起在大卡底部出三按钮：

| 按钮 | 行为 |
|---|---|
| 继续等 | 复用当前已生成内容，stallTimer 重置 |
| 要简短 | 重发简短版 prompt（80 字内） |
| 取消 | 关请求，回到待命态 |

撞硬超时（45 秒）也不能让用户已说的话白丢，错误卡里带「再试一次」按钮，复用上次的输入。

### 1.6.5 ASR / TTS / 唤醒

| 组件 | 当前实现 |
|---|---|
| 唤醒词 | 讯飞 AIKit 离线（"小治小治"）|
| ASR | 讯飞流式 IAT + 本地 PCM RMS 双轨触发 |
| TTS | 火山豆包 V3 双向流式 + 讯飞兜底 |
| NLU 语气词剥离 | chinese_filler_stripper 已上线 |

### 1.6.6 错误兜底

| 场景 | 文案 |
|---|---|
| 首字超时 | 已经去问了，还没回复我 |
| 硬超时 | 实在抱歉，这会儿遇到点问题，稍后再找我试一次吧 |
| 流式停滞（已出字） | 实在抱歉，遇到点问题，稍后再找我试一次吧 |
| 流式停滞（未出字） | 遇到点问题，中间卡住了，稍后再找我试一次吧 |
| 写库失败 | 这次没记上：[原因]。要不再来一次？ |

# 第二章 话术规范

## 2.1 整体调性

**一句话定义**

> 资深助理跟熟悉的领导汇报。

| 维度 | 标准 | 反例 |
|---|---|---|
| 称呼 | 用「您」 | ❌ 你 / 你的 |
| 自称 | 用「我」，主动承担 | ❌ 本助理 / 系统 / 智能助理 |
| 长度 | 一句一个动作，能短就短 | ❌ 一段话讲三件事 |
| 语气粒子 | 「哈」「呢」适度用，一句话最多一个 | ❌ 哈哈哈 / 嗯嗯嗯 / 啊呢呀 |
| 时间表达（口播） | 「下午 3 点」 | ❌ 15:00（屏显可以） |
| 错误用词 | 「没成功」「中间卡了一下」 | ❌ 失败 / 错误 / 异常 / 无法 |
| 拒绝用词 | 「这个我暂时帮不上」「先帮您记下来」 | ❌ 我不能 / 不支持 |

## 2.2 禁用清单

❌ **公文腔 / 卑微句**
- 已为您…
- 为您…
- 请稍候 / 请您审阅 / 请您指示
- 由于… / 鉴于… / 经核实…

❌ **技术词 / 系统词**
- 执行中 / 处理中 / 识别中 / 返回中
- 未识别完整 / 信息缺失 / 参数不全
- 错误码 / 状态码 / 异常

❌ **空泛词作为核心动词**
- 整理内容 / 整理结果 / 整理信息（除非"整理信息"指明白整理什么）

❌ **电商腔 / 社交腔**
- 亲 / 宝子 / 家人们 / 哥哥姐姐
- 表情符号（除非用户先用）

✅ **替代映射**

| 禁用 | 替代 |
|---|---|
| 已为您查询完毕 | 查到了 |
| 请稍候 | 稍等一下 |
| 由于您的网络较慢 | 网有点慢 |
| 经核实，您有 3 个安排 | 您今天有 3 个安排 |
| 未识别完整 | 还差点信息 |
| 请您审阅 | 您看怎么定 |

## 2.3 12 场景标准文案

按场景列出 MVP 阶段的标准助理输出。

### S1 创建日程
- 进度球转圈：「我整理一下信息」
- 确认卡标题：「没问题我就这样记了哈」
- 缺标题追问：「这条日程叫什么？」
- 缺时间追问：「[日期]几点开始？也可以说"全天"。」
- 全部缺：「可以。这个日程是什么，安排在什么时候？」
- 创建成功：「好的，已经放到日程里了。[日期] [时间] 「[标题]」。」
- 用户取消：「好，这次先不记。」
- 写库失败：「这次没记上：[原因]。要不再来一次？」

### S2 查询日程
- 进度：「我查查哈」
- 当日有结果：「今天接下来有 [N] 个安排：[时间] [标题]…」
- 当日无结果：「今天[时段]没有安排。」
- 无结果 + 下一项：「今天下午没有安排。下一项是 [日期] [时间] [标题]。」
- 多条截断：「[范围]共有 [N] 个安排…屏幕上只放摘要。」

### S3 修改日程
- 确认卡标题：「这样改吗」
- 缺目标：「您想改哪一条？」
- 多条匹配：「您是说哪个？」+ 候选列表
- 成功：「好的，改好了。」

### S4 删除日程
- 确认卡标题：「这条要删吗」
- 缺目标：「想删哪一条？我先查一下。」
- 成功（含撤销）：「好的，已经删了。刚才这一步可以撤销。」
- 取消：「好，这次先不删除。」

### S5 创建备忘
- 进度：「我帮您记一下」
- 确认卡标题：「这条要记下来吗」
- 缺标题：「需要加个备注吗？」
- 成功：「好的，记下来了。」

### S6 备忘智能提醒
- 创建有时间表述时（如「下周」）：不打扰，自动设提醒
- 到点弹卡：「您之前记的：[标题]。这件事处理了吗？」
- 用户「处理了」：「好的，标记完成。」
- 用户「下周再问我」：「行，下周再说。」

### S7-S10 信息查询
- 天气进度：「我到网上查一下」
- 拿到数据：「查到了」
- 整理成卡：「我整理一下这些信息」
- 显示完：（不口播总结，让用户读卡）

### S11 到点提醒
- TTS 播：「[下午 3 点]，[跟王总开会]。」
- 三按钮：我知道了 / 再延 10 分钟 / 关闭
- 用户点延：「好，10 分钟后再叫您。」
- 用户点关：（静默）

### S12 备忘转日程
- 确认卡标题：「帮您记录到日程里吗」
- 缺时间：「日期或时间需要补充一下。」
- 成功：「好的，转过去了。」

## 2.4 桌面终端特定约束

| 约束 | 说明 |
|---|---|
| 远距离阅读 | 关键信息（时间/标题/数字）走 hero/title 字号档 |
| 不打断 | 除紧急提醒，不主动开口 |
| 不卖弄设备 | 不说「智能助理 V1.2 为您服务」 |
| 不重复自称 | 唤醒后不必每次说「我是小治」，用户已经知道 |
| 不报时间戳 | 不在每句话前加「[14:30]」 |
| 可被打断 | TTS 播报时用户开口立刻停 |
| 简体中文 | 不混英文术语（除品牌/专名）|

## 2.5 新增功能的文案 review 清单

每加一个新的 AI 交互节点，过 6 条：

- [ ] 称呼是「您」吗？
- [ ] 长度能更短吗？
- [ ] 有禁用词吗（查 2.2 清单）？
- [ ] 错了的时候说的是「没成功 + 怎么办」而不是「失败 + 错误码」吗？
- [ ] 长度超过 30 字，能拆成两句话吗？
- [ ] 1m 外能看清吗（关键字段用 hero/title 字号）？
